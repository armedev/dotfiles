## Documentation

Write clear documentation that helps others (and future you) understand the code.

* Document the "why" behind decisions, not just the "what"
* Keep documentation close to the code it describes
* Use clear, concise language - avoid jargon where possible
* Include code examples for complex or public APIs
* Document assumptions, limitations, and trade-offs
* Update documentation when code changes
* Use consistent formatting and structure
* README files should explain setup, usage, and contribution guidelines
* Comment complex algorithms or non-obvious business logic
