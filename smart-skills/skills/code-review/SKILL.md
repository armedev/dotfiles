## Code Review

Review code thoroughly to catch issues early and maintain quality standards.

* Verify the code solves the intended problem correctly
* Check for edge cases and error conditions
* Ensure error handling is robust and informative
* Review naming - variables, functions, and types should be descriptive
* Look for security vulnerabilities (injection, XSS, etc.)
* Verify tests exist and cover the changes adequately
* Check for code duplication and opportunities to refactor
* Ensure consistent style with the existing codebase
* Consider performance implications of the implementation
* Ask questions if something is unclear or seems odd
