## Error Handling

Handle errors gracefully to create robust and user-friendly applications.

* Fail fast with clear, actionable error messages
* Handle errors at the appropriate level of abstraction
* Distinguish between recoverable errors and fatal failures
* Never swallow exceptions or ignore errors silently
* Provide context in error messages - what went wrong and why
* Use specific error types rather than generic exceptions
* Log errors with sufficient detail for debugging
* Present user-friendly messages to end users
* Implement proper cleanup in error scenarios
* Consider retry logic for transient failures
