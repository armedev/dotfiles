## Git Workflow

Use Git effectively to track changes and collaborate with others.

* Commit early and commit often - small commits are easier to review
* Write clear, descriptive commit messages explaining the "why"
* Use present tense and imperative mood (e.g., "Add feature" not "Added feature")
* Keep commits focused on a single logical change
* Use feature branches for new work, keep main/master stable
* Pull and rebase regularly to avoid merge conflicts
* Review changes with `git diff` before committing
* Use meaningful branch names (e.g., `feature/user-authentication`)
* Never commit secrets, passwords, or sensitive data
