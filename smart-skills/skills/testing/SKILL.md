## Testing

Write comprehensive tests to ensure code works correctly and remains stable.

* Write tests before or alongside implementation code (TDD when possible)
* Test the happy path, edge cases, and error scenarios
* Keep tests independent - no shared state between tests
* Make tests fast and deterministic
* Use descriptive test names that explain what's being tested
* Mock external dependencies to test in isolation
* Aim for meaningful coverage, not just high percentages
* Update tests when requirements or implementation changes
* Use property-based testing for complex input validation
